# 尖塔扫雷

[<kbd>中文</kbd>](./README.md) [<kbd>English</kbd>](./README_EN.md)

一个带有角色成长、金币收益和道具构筑的网页扫雷小游戏。玩家在不同规模的矿区中排雷，通过胜利获得金币与经验，再用金币升级角色、购买永久道具，逐步解锁高风险高收益的关卡。

## 游戏特色

- 经典扫雷操作：左键翻开格子，右键插旗标记地雷。
- 四档难度：简单、进阶、困难、终极，每档拥有不同棋盘大小、地雷数量、目标时间和收益权重。
- 成长系统：胜利获得金币和经验，升级后提高结算倍率，最高 20 级。
- 商店系统：金币可购买永久道具，道具会在后续对局中自动或主动生效。
- 时间收益：通关越快，时间系数越高，最终金币越多。
- 本地存档：等级、经验、金币、已购买道具会保存到浏览器 `localStorage`。

## 如何运行

这是一个纯前端项目，不需要构建步骤。

1. 进入项目目录。
2. 用浏览器打开 `index.html`。
3. 开始游戏。

也可以使用任意静态文件服务器运行项目，例如 VS Code Live Server。

## 玩法说明

游戏开始后，中央区域会生成扫雷棋盘。玩家需要翻开所有安全格，避开地雷。

- 左键：翻开格子。
- 右键：插旗或取消插旗。
- 新一局：根据当前选择的难度重新开始。
- 升级：消耗金币提升等级，并提高等级倍率。
- 购买道具：商店中的道具为永久购买，购买后会在每局中生效。
- 重置存档：清空当前浏览器中的等级、金币和道具数据。

第一次点击会进行一定保护，避免开局直接踩雷；购买特定道具后，首点安全区域还会进一步扩大。

## 结算规则

基础金币公式：

```text
金币 = 120 × 难度权重 × 时间系数 × 等级倍率 × 道具倍率
```

其中：

```text
时间系数 = 目标时间 / 实际结算时间
```

时间系数最低为 `0.35`，最高为 `2.50`。部分道具可以减少结算用时、提高金币倍率，或针对困难和终极难度提供额外收益。

经验收益与难度权重相关，终极难度在购买终局道具后还可以获得额外经验加成。

## 道具系统

商店道具按照等级逐步开放，分为几类：

- 护命：踩雷时免死，或提供额外容错。
- 主动：在棋盘上方显示使用按钮，可揭示安全格、标记地雷、窥视未知格等。
- 开局：每局开始时自动翻开安全格、移除部分地雷或增强首次点击。
- 收益：提高胜利金币，或在特定难度和时间条件下追加倍率。
- 结算：减少实际结算用时，让速度收益更容易触发。
- 终局：强化终极难度的金币与经验收益。

## 代码概览

- `index.html` 定义顶部玩家状态栏、左侧难度选择、中间棋盘区域和右侧商店面板。
- `styles.css` 负责整体视觉风格、三栏布局、棋盘格、道具卡片和移动端适配。
- `game.js` 包含主要游戏逻辑：
  - 难度和道具配置。
  - 本地存档读取与写入。
  - 棋盘生成、邻格计算、开格、插旗、胜负判断。
  - 主动道具使用逻辑。
  - 金币、经验、等级和收益倍率结算。

## 当前状态

项目目前是一个轻量级单页游戏原型，所有逻辑都集中在前端文件中。适合继续扩展更多道具、成就、音效、动画、排行榜或关卡目标。
